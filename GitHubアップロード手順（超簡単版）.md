# GitHubにファイルをアップロードする手順（超簡単版）

## 🎯 目標
ローカルのフォントファイルをGitHubにアップロードして、Figma Makeで使えるようにする

---

## 📝 ステップ1: GitHub Desktopを開く

1. GitHub Desktopアプリを開く
2. 左側のリストから「LINE-Seed-JP」というリポジトリを選ぶ

---

## 📝 ステップ2: ファイルを確認する

1. GitHub Desktopの画面で「Changes」タブをクリック
2. 左側に「Uncommitted changes」というセクションがあるはず
3. そこにたくさんのファイルが表示されているはずです：
   - `fonts` フォルダ
   - `README.md`
   - その他のファイル

**もし何も表示されていない場合**：
- 画面の下の方に「No local changes」と表示されているかもしれません
- その場合は、すでにコミット済みかもしれません（ステップ3に進んでください）

---

## 📝 ステップ3: コミットする（ファイルを保存する準備）

1. 画面の左下に「Summary (required)」という入力欄があります
2. そこに「Add font files」と入力（何でもOK、変更内容を説明するメモ）
3. その下の「Description」は空欄でOK
4. 「Commit to main」という青いボタンをクリック

**これでファイルが「コミット」されました**（まだGitHubにはアップロードされていません）

---

## 📝 ステップ4: GitHubにアップロードする（プッシュ）

1. コミットが完了すると、画面の上に「Push origin」というボタンが表示されます
2. そのボタンをクリック
3. 少し待つと、GitHubにアップロードされます

**もし「Push origin」ボタンが見つからない場合**：
- 画面の上の方、右側に「Publish branch」というボタンがあるかもしれません
- それをクリックしてください

---

## 📝 ステップ5: 確認する

1. ブラウザでGitHubのページを更新（F5キーまたは⌘+R）
2. `fonts`フォルダが表示されているはずです！

---

## ❓ よくある質問

### Q: 「Changes」タブに何も表示されない
A: すでにコミット済みかもしれません。画面の上の方に「Push origin」ボタンがないか確認してください。

### Q: 「Push origin」ボタンが見つからない
A: 画面の上の方、右側に「Publish branch」というボタンがあるかもしれません。それをクリックしてください。

### Q: エラーが出た
A: エラーメッセージの内容を教えてください。一緒に解決しましょう。
